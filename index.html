The Negative Boy

by: Isis Jenkins

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I chose to make a story based game instead of the more commonly seen game designs. I appreciate a strong story line. A few years ago my mom and I ran across a published game with a unique plan in content delivery. As intriguing as it was, we werent quite able to get the whole picture. When I got the opportunity to make my own game, I chose this style telling myselfif I cant beat their game, I guess Ill have to beat mine.

2. What I Learned

My teachers feedback brought conflicting features to my attention, such as adding a color changing effect into my program while maintaining the way Ive written my code. User testers and classmates gave me an insight to how user friendly my are was while running. Tweaking minor details, to make playing less confusing, was well needed.

3. Architecture of My Game

Although I have minimal classes, each class is dependent on each other. Where one class builds the cosmetics of the story page in my game, another builds the map, while another holds information. My story page doesnt know what to display until the map page identifies where the player is (in relation to the map) and then goes to retrieve the proper dialogue to feed back to the story page.

4. Technical Implementation

I am very proud of the map I created which includes the navigation tools and constant recognition of the players location. Using an array I was able to get the basic layout, essentially making a grid. However, the grid didnt do anything until I created methods that allowed me to set the user into a beginning post and identify it. To get the navigation buttons working I had to make requirements that checked the condition of each variable in the array and update the map. Then to rap it all up, I assigned symbols to the grid so that the user can recognize an area that theyve already been on. I also like my death roll which randomly selects if the player dies or not, allowing a different amount of deaths each time you play the game.

5. Most Challenging Part

I think the most challenging part of my game was understanding how to accurately and efficiently tie my classes together. I came to points where I may place code in a class that didnt suit its needs, or place commands in areas that made my code redundant and messy. I often found myself searching for a complex way of reaching my goal not noticing that the answer was actually simple, clean, and productive.

6. Future Fixes and Updates

If I were to continue working in this game I would like to make the screen gradually fade to the color red when the player dies. Id also make the game a little longer with a bigger story and map because the current play time of the game is fit in length so that the player can finish the game in one sitting.
